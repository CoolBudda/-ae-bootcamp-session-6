# Quickstart: Validate "Support for Overdue Todo Items"

This guide validates the feature end-to-end against the spec. Implementation details live in
`tasks.md` (generated by `/speckit.tasks`) and the source files; this is a run/validation
guide only.

## Prerequisites

- Node.js v16+ and npm v7+
- Dependencies installed: `npm install` (from repo root)

## Run the automated tests

The feature is covered by unit tests for the `isOverdue` helper and component tests for the
overdue indicator in `TodoCard`.

```bash
# All frontend tests
npm test --workspace=packages/frontend

# Just the relevant suites
npm test --workspace=packages/frontend -- dateUtils.test.js
npm test --workspace=packages/frontend -- TodoCard.test.js
```

Expected: all suites pass; coverage for the new helper and the overdue rendering path is
complete (per Testing Guidelines, 80%+ overall, 100% on this critical path).

Contract reference: [contracts/isOverdue.md](./contracts/isOverdue.md). Data rules:
[data-model.md](./data-model.md).

## Manual validation

```bash
npm run start   # from repo root (starts frontend on :3000, backend on :3030)
```

Then, in the app:

1. **Overdue shown (US1 / FR-001, FR-005, FR-006)**: Add an incomplete todo with a due date in
   the past. Confirm it is visually distinguished AND shows a non-color cue (e.g., an
   "Overdue" label/icon), not color alone.
2. **Due today not overdue (FR-004)**: Add an incomplete todo due today. Confirm it is NOT
   marked overdue.
3. **Future not overdue (FR-001)**: Add an incomplete todo due tomorrow. Confirm it is NOT
   marked overdue.
4. **No due date (US2 / FR-003)**: Add an incomplete todo with no due date. Confirm it is NOT
   marked overdue.
5. **Completed excluded (US2 / FR-002, FR-009)**: Mark an overdue todo complete. Confirm the
   overdue indication disappears immediately.
6. **Edit clears/sets overdue (FR-009)**: Edit an overdue todo's due date to today or a future
   date — indicator clears. Edit a future todo's due date to the past — indicator appears.
7. **Accessibility (FR-006, FR-007)**: Toggle dark/light mode and confirm the indicator is
   legible with sufficient contrast in both, and is perceivable without relying on color.

## Success criteria mapping

- SC-001: Steps 1–5 confirm only incomplete, past-due, dated todos are flagged.
- SC-002: Step 1 — overdue items are identifiable at a glance.
- SC-003: Step 7 — non-color cue + WCAG AA contrast in both themes.
- SC-004: Steps 5–6 — indicator removed immediately on completion/date change.
